# 我的菜谱 — 产品需求文档 (PRD)

> 版本：v1.0 | 最后更新：2026-06-23

---

## 1. 产品概述

### 1.1 产品定位

一款面向个人/家庭的**菜谱管理与购物清单工具**，帮助用户管理自己会做的菜品，在做饭前快速规划菜单、合并食材清单，提高买菜备菜效率。

### 1.2 核心价值

| 用户痛点 | 解决方案 |
|---------|---------|
| 会做的菜很多，但每次想不起来吃什么 | 分类浏览 + 搜索，快速浏览所有菜谱 |
| 买菜时漏买食材、多买重复 | 多道菜自动合并购物清单，标注重复食材 |
| 菜谱散落在各个 App/网页/笔记中 | 统一管理，支持网页自动解析和文字导入 |
| 冰箱有食材但不知道能做什么 | 按已有食材搜索可做的菜 |

### 1.3 目标用户

- **独居/小家庭**：管理日常家常菜谱，规划每周菜单
- **做饭爱好者**：收集和整理自己的拿手菜
- **厨房新手**：跟着自己的菜谱买菜做饭，减少试错

## 2. 功能需求

### 2.1 菜谱管理（核心）

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 添加菜谱 | 手动填写菜名、食材、标签、备注 | P0 |
| 编辑菜谱 | 修改已有菜谱的全部信息 | P0 |
| 删除菜谱 | 删除不需要的菜谱 | P0 |
| 菜谱详情 | 展示菜名、食材清单、备注 | P0 |
| 封面图 | 上传/粘贴菜品成品图，居中裁剪 3:2 | P1 |

### 2.2 分类与浏览

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 主分类 | 荤菜/素菜/海鲜/汤羹/主食/甜品 | P0 |
| 子分类 | 每个主分类下的子类别，如猪肉/牛肉/鱼类 | P1 |
| 分类自定义 | 用户可增删改主分类和子分类 | P1 |
| 左右分栏浏览 | 左侧固定标签栏 + 右侧瀑布流菜谱卡片 | P0 |
| 搜索入口 | 分类页右上角搜索图标，点击展开搜索框 | P0 |

### 2.3 搜索

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 按菜名搜索 | 模糊匹配菜名和标签 | P0 |
| 按食材搜索 | 输入已有食材，返回可做的菜 | P0 |

### 2.4 购物清单

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 加入待买清单 | 从详情页或分类页将菜谱加入清单 | P0 |
| 菜谱勾选开关 | 勾选的菜谱才计入合并清单 | P0 |
| 自动合并食材 | 多道菜同种食材自动合并数量 | P0 |
| 食材来源标注 | 每项食材标注来自哪些菜谱 | P0 |
| 食材计数 | 跨菜谱出现的食材显示 ×N | P1 |
| 调料过滤 | 油盐酱醋葱姜蒜等自动过滤不显示 | P0 |
| 食材排除 | 删除不想要的食材（重新勾选菜谱可恢复） | P1 |
| 勾选已购 | 买到后勾选标记 | P1 |

### 2.5 导入功能

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 网页链接导入 | 粘贴链接，自动解析菜谱信息和食材 | P1 |
| 文字解析导入 | 粘贴任意格式菜谱文字，自动识别菜名、主料、调料、步骤 | P1 |
| 主料/调料自动分离 | 按内置分类表自动区分食材和调料 | P1 |
| 步骤自动编号 | 按动作关键词自动拆分步骤并编号 | P1 |
| 编辑后导入 | 解析结果支持编辑修改后再保存 | P2 |

### 2.6 标签与备注

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 自定义标签 | 自由添加标签，如"快手菜"、"下饭菜" | P1 |
| 备注 | 自由文本，记录心得、注意事项 | P1 |

### 2.7 数据管理

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 本地存储 | localStorage 持久化，离线可用 | P0 |
| 多设备同步 | Node.js 共享服务器，局域网多设备同享数据 | P1 |
| 导出/导入 | JSON 格式导出备份，剪贴板导入恢复 | P2 |
| 清空数据 | 一键清空所有菜谱数据 | P2 |

## 3. 技术架构

### 3.1 前端技术栈

| 层 | 技术 |
|----|------|
| 框架 | 原生 HTML/CSS/JavaScript |
| 存储 | localStorage（降级方案）+ 内存缓存 |
| 样式 | 自定义 CSS（Apple 设计风格） |
| 图标 | SVG → PNG（sharp 转换） |
| 打包 | 纯静态，无需构建工具 |

### 3.2 同步方案

```
浏览器 → Node.js 服务器（port 3456）→ shared_data.json
   ↑                     ↓
localStorage ← 离线降级 文件持久化
```

- 有服务器时：读写走 API，自动同步到所有设备
- 服务器不可用时：自动降级到 localStorage
- 数据隔离：每台设备各自独立的 localStorage 作为本地缓存

### 3.3 运行方式

```bash
# Web 版（推荐）
cd web && node server.js
# 访问 http://localhost:3456

# 微信小程序版（备选）
# miniprogram/ 目录，微信开发者工具打开
```

## 4. 数据模型

### 4.1 菜谱 (Recipe)

```typescript
interface Recipe {
  id: string;              // UUID
  name: string;            // 菜名（必填）
  category: string;        // 主分类 key
  subcategory: string;     // 子分类
  coverImagePath: string | null;  // 封面图 base64
  ingredients: Ingredient[];  // 食材清单
  tags: string[];          // 标签
  notes: string;           // 备注
  createdAt: number;       // 创建时间戳
  updatedAt: number;       // 更新时间戳
}
```

### 4.2 食材 (Ingredient)

```typescript
interface Ingredient {
  id: string;
  canonicalName: string;   // 标准化名称（如"鸡蛋"）
  amount: string;          // 数量文本（如"2个"）
  quantity: number | null; // 解析后的数值
  unit: string | null;     // 标准化单位
  notes: string;           // 备注
}
```

### 4.3 合并清单项 (MergedIngredient)

```typescript
interface MergedIngredient {
  name: string;            // 食材名称
  unit: string | null;
  qty: number | null;      // 合并后的总数量
  rawAmounts: string[];    // 原文数量列表
  recipes: { rid: string; rname: string; txt: string }[];  // 来源菜谱
  checked: boolean;        // 是否已采购
}
```

## 5. 用户界面

### 5.1 页面结构

| 页面 | 说明 | 入口 |
|------|------|------|
| 首页 | 统计卡片 + 分类概览 + 最近添加 | Tab 1 |
| 分类浏览 | 左标签栏 + 右卡片流 | Tab 2 |
| 待买清单 | 已选菜谱 + 合并食材清单 | Tab 3 |
| 我的 | 数据统计 + 功能入口 + 存储管理 | Tab 4 |
| 菜谱详情 | 菜名 + 食材 + 备注 | 点击菜谱卡片 |
| 添加/编辑菜谱 | 表单：菜名 + 子分类 + 食材 + 标签 + 备注 | FAB / 编辑按钮 |
| 导入 | 粘贴链接或文字，自动解析 | 我的 / FAB |

### 5.2 交互流程

```
用户打开 App
  → 首页看到统计和最近菜谱
  → 点击分类 Tab → 左栏选分类 → 右栏浏览菜谱
  → 点击菜谱 → 看详情 → 加入待买清单
  → 切换待买清单 Tab → 勾选菜谱 → 看合并食材
  → 到超市按清单买菜 → 勾选已购
```

### 5.3 设计风格

- Apple 设计语言：SF Pro 字体、毛玻璃导航栏、iOS 蓝 (#007AFF)
- 响应式布局：max-width 430px，适配手机竖屏
- 毛玻璃效果：`backdrop-filter: blur(20px)`
- 圆角框架：8px 基础圆角，24px 胶囊按钮

## 6. 依赖与限制

### 6.1 外部依赖

| 依赖 | 用途 | 替代方案 |
|------|------|---------|
| allorigins.win | 网页抓取跨域代理（可选） | 自建代理 / 手动录入 |

### 6.2 限制

- 存储上限：localStorage 约 10MB，约可存 100+ 菜谱（含压缩封面图）
- 同步依赖：共享服务器需局域网内运行，不支持公网访问
- 图片：封面图压缩至 40KB 以内（800px 宽，JPEG quality 0.7）
